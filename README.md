## 🧠 Arquitetura e Lógica do Sistema

O **Neural Sentinel** opera sob uma lógica de **Dados Sintéticos (Synthetic Data)** para simular o tráfego de rede global. 

### 📦 Analogia: Bolinhas e Caixas
1.  **A Caixa Principal:** É o `st.session_state`, que mantém o estado da aplicação entre os refrescos de tela.
2.  **As Bolinhas (Eventos):** São geradas randomicamente e classificadas como `INFO` (Tráfego Normal) ou `ALERTA` (Tentativa de Invasão).
3.  **O Funil (Filtro):** Implementamos um sistema de filtragem na **Barra Lateral (Sidebar)** que decide quais bolinhas serão exibidas no **Log de Eventos** sem perder a integridade dos dados brutos.

## ⚡ Funcionalidades Principais (Key Features)

* **Monitoramento Global:** Globo 3D interativo com rotação dinâmica.
* **Filtro de Severidade:** Seleção em tempo real de logs críticos ou informativos.
* **Telemetria em Tempo Real:** Gráficos de barras dinâmicos monitorando a carga dos servidores mundiais.
* **Sistema de Alerta Visual:** Interface muda de cor (`Neon Blue` para `Cyber Red`) durante detecções de ameaças.
